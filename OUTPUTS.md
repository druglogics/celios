# CELIOS Output Files

Complete reference for all output files generated by CELIOS.

---

## Output File Overview

CELIOS generates outputs in three stages:

| Step | Output | File | Format |
|------|--------|------|--------|
| 1 | Node Dictionary | `node_HGNC_dict.csv` | CSV |
| 2 | Cell-Line Mapping | `identifiers.csv` | CSV |
| 3 | Master Activity Matrix | `activity_master_matrix.csv` | CSV |
| 3 | Priority-Selected Activity | `activity_from_master.csv` | CSV |
| 3 (opt) | Per-Cell-Line Files | `cell_lines/*.csv` | CSV |
| 3 (opt) | Tissue-Organized Files | `tissue_folders/<Tissue>/<Cell>/*.csv` | CSV |
| All | Execution Log | `run_log.txt` | TXT |

---

## Step 1: Node Dictionary

### Output File

**File:** `node_HGNC_dict.csv`

**Location:** `<output_directory>/`

**Format:** CSV with 2 columns

```
node_name,symbols
TP53,"TP53, P53"
BRCA1,BRCA1
PARP1,"PARP1, PARP, ARTD1"
```

### Column Description

- **node_name** (str): Node identifier from network (SIF file)
- **symbols** (str): Comma-separated HGNC gene symbols for this node

### How It's Generated

1. Parse biological network (SIF format)
2. Extract unique nodes from network
3. Map each node to HGNC gene symbols using reference file
4. Apply manual corrections (if provided)
5. Save to CSV

### Downstream Use

- **Input to Step 3:** Defines which nodes appear in final activity matrix
- **Quality check:** Verify mappings are correct before continuing
- **Skip Step 1:** Provide pre-built dictionary to start at Step 2

### Example Interpretation

```
TP53 → "TP53, P53"
```
Means: Network node "TP53" maps to HGNC symbol "TP53" (with alias "P53")

## Step 2: Cell-Line Mapping

### Output File

**File:** `identifiers.csv` (internal reference)

**Location:** `<output_directory>/`

**Format:** CSV mapping SIDM → cell line metadata

```
tissue,SIDM,cell_line_name
breast,SIDM00001,MCF7
lung,SIDM00002,H1299
colon,SIDM00003,HCT116
```

### Use

- Internal file for tissue organization
- Maps SIDM identifiers to cell line names
- Used when organizing outputs by tissue

### When Generated

- Always created when Activity step runs
- Generated from `cell_line_file` config

## Step 3: Activity Matrices

### Master Activity Matrix

**File:** `activity_master_matrix.csv`

**Location:** `<output_directory>/`

**Purpose:** View all omics data sources combined

**Format:** CSV

```
node,MCF7__mutations,MCF7__cnv,MCF7__TF,MCF7__expression,H1299__mutations,H1299__expression
TP53,1,0,0.5,2.3,0,1.2
BRCA1,0,1,0.8,3.1,1,2.5
PARP1,0,0,0.6,1.5,0,1.8
```

### Structure

- **Index/First column:** Node names (from node dictionary)
- **Columns:** `{cell_line}__{data_source}` pattern
- **Values:**
  - Binary data (mutations, CNV): 0 or 1
  - Continuous data (TF, expression): numeric scores

### Data Source Columns

For each cell line, included columns depend on config:

- `{cell}__mutations` - Binary mutation status
- `{cell}__cnv` - Copy number variation (binary)
- `{cell}__TF` - TF activity (continuous)
- `{cell}__expression` - Gene expression (continuous)

### Use Cases

- **Exploratory analysis:** See all data sources together
- **Debugging:** Identify data quality issues
- **Custom ranking:** Use different priority than default
- **Integration studies:** Analyze multi-source agreement

### Example Interpretation

Row `TP53`:
- MCF7 has mutation (1), no CNV (0), moderate TF activity (0.5), high expression (2.3)
- H1299 has no mutation (0), high expression (1.2)

---

### Priority-Selected Activity

**File:** `activity_from_master.csv`

**Location:** `<output_directory>/`

**Purpose:** Ready for downstream analysis (Boolean model calibration)

**Format:** CSV

```
node,MCF7,H1299,HCT116
TP53,1,0,1
BRCA1,1,1,1
PARP1,0,1,1
```

### Structure

- **Index/First column:** Node names
- **Columns:** Cell line names
- **Values:** Single activity value per node-cell pair

### How Values Are Selected

Each cell (node × cell_line) contains the value from the **highest-priority data source available**:

1. Check mutations → use if available (1 or 0)
2. Else check CNV → use if available
3. Else check TF activity → use if available
4. Else use expression → use if available
5. Else: NaN (no data)

**Default priority:** `["mutations", "cnv", "TF", "expression"]`

**Customizable:** Set `data_sources` in config to change order

### Example

For node `TP53` in cell line `MCF7`:
- Master matrix: mutations=1, cnv=0, TF=0.5, expression=2.3
- Selected value: **1** (from mutations, highest priority)

### Use Cases

- **Boolean models:** Direct input to model calibration
- **Downstream analysis:** Statistical testing, visualization
- **DrugLogics:** Ready for training file generation

## Step 3: Per-Cell-Line Training Files

### Legacy Mode (cellfiles_dir)

**Location:** `<cellfiles_dir>/<cell_line_name>.csv`

**Example:**
```
results/cell_lines/MCF7.csv
results/cell_lines/H1299.csv
```

**Format:** CSV, one file per cell line

```
node,activity
TP53,1
BRCA1,0
PARP1,1
```

**Use:** One file per cell line, compatible with DrugLogics pipeline

---

### Tissue-Organized Mode (tissue_dir)

**Location:** `<tissue_dir>/<Tissue>/<CellLine>/training.csv`

**Directory Structure:**

```
tissue_folders/
├── breast/
│   ├── MCF7/
│   │   └── training.csv
│   ├── T47D/
│   │   └── training.csv
├── lung/
│   ├── H1299/
│   │   └── training.csv
│   └── H460/
│       └── training.csv
└── colon/
    └── HCT116/
        └── training.csv
```

**Format:** Same as legacy (nodes × activity), organized by tissue/cell_line

**Use:** When you need tissue-based file organization

**Enabling:** Set `paths.tissue_dir` in config + provide `tissue` column in `cell_line_file`

- **Warnings/Errors:** Any issues encountered (with recovery info)

- **Status: ✓ COMPLETE** = Step succeeded
- **Status: ✗ FAILED** = Step failed (details above)
- **% values:** Data coverage/quality metrics
- **Unmapped nodes:** Nodes that couldn't be mapped to HGNC symbols

## Handling Missing Data

### What If a Cell Line Has No Data?

If a cell line is missing from a data source:
- **Master matrix:** Column will have NaN values
- **Final matrix:** Falls back to next priority source (or NaN if all sources missing)
- **Log:** Reported in "Data coverage" section

### What If a Node Is Not in Output?

Possible causes:
- Node not in node dictionary (verify Step 1 output)
- Node not in activity file (verify data is present)
- Node filtered out by format detection (check log)

### Verifying Output Quality

```bash
# Check node dictionary
head node_HGNC_dict.csv
wc -l node_HGNC_dict.csv

# Check final activity matrix
head activity_from_master.csv
# Check for missing values
grep -c ",,\|,$" activity_from_master.csv

# Check execution log
tail -20 run_log.txt
```

## Output Locations

### Default Structure

```
<output_directory>/
├── node_HGNC_dict.csv              # Step 1 output
├── identifiers.csv                 # Step 2 output
├── activity_master_matrix.csv      # Step 3 output
├── activity_from_master.csv        # Step 3 output
├── run_log.txt                     # Execution log
├── cell_lines/                     # Optional: per-cell-line files
│   ├── MCF7.csv
│   ├── H1299.csv
│   └── ...
└── tissue_folders/                 # Optional: tissue-organized
    ├── breast/
    │   ├── MCF7/
    │   │   └── training.csv
    │   └── ...
    └── lung/
        └── ...
```

### Configuration Options

- `paths.output` - Main output directory (required)
- `paths.cellfiles_dir` - Per-cell-line files location (optional)
- `paths.tissue_dir` - Tissue-organized files location (optional)

### Customizing Output Location

```yaml
paths:
  output: "results/analysis_v2"        # All outputs here
  cellfiles_dir: "results/training"    # Per-cell-line files
  tissue_dir: "results/by_tissue"      # Tissue organization
```

## Using Outputs in Downstream Analysis

### 1. Boolean Model Calibration (DrugLogics)

Use `activity_from_master.csv` or per-cell-line files:
```bash
cd results/tissue_folders
# Use CSV files directly in DrugLogics pipeline
```

### 2. Statistical Analysis

```python
import pandas as pd

# Load final activities
activities = pd.read_csv("activity_from_master.csv", index_col=0)

# Analyze
print(f"Nodes: {len(activities)}")
print(f"Cell lines: {len(activities.columns)}")

# Identify cell-line specific mutations
mutations = pd.read_csv("activity_master_matrix.csv", index_col=0)
mut_cols = [c for c in mutations.columns if '__mutations' in c]
print(f"Highly mutated nodes: {mutations[mut_cols].sum(axis=1).nlargest(10)}")
```

### 3. Custom Integration

```python
# Load master matrix for custom analysis
import pandas as pd

master = pd.read_csv("activity_master_matrix.csv", index_col=0)

# Get specific data source
mutations = master[[c for c in master.columns if '__mutations' in c]]
expression = master[[c for c in master.columns if '__expression' in c]]

# Custom analysis
print(f"Mutation coverage: {mutations.notna().sum().sum() / mutations.size:.1%}")
```

## Troubleshooting Outputs

| Issue | Cause | Check |
|-------|-------|-------|
| Empty activity matrices | No data in input files | Check run_log.txt data coverage |
| Missing cell lines | Format parsing failed | Verify input file format |
| Node count mismatch | Nodes not in activity file | Check node_HGNC_dict.csv vs input files |
| All NaN in final matrix | All sources missing for nodes | Check master matrix for coverage |
| Files not generated | Step failed silently | See run_log.txt for errors |

For detailed troubleshooting, see [USAGE.md](USAGE.md#troubleshooting).
